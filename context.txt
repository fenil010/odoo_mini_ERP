# Mini ERP System

## From Demand to Delivery

---

# Project Overview

Mini ERP is a web-based Enterprise Resource Planning system designed for manufacturing businesses.

The goal is to connect:

* Products
* Inventory
* Sales
* Purchase
* Manufacturing
* Procurement
* Audit Logs

into a single centralized platform.

Instead of managing operations through spreadsheets and manual processes, the ERP automates stock tracking, manufacturing planning, procurement decisions, and order fulfillment.

# Why Next.js?

We chose Next.js because it provides:

* Frontend and Backend in one codebase
* Server Components
* Server Actions
* Fast development speed
* Easy deployment
* Excellent TypeScript support

For a hackathon this reduces development time significantly.

---

# Why PostgreSQL?

We chose PostgreSQL because:

* Strong relational support
* ACID compliance
* Foreign Keys
* Transaction support
* Complex reporting queries
* Ideal for ERP systems

ERP applications have many relationships:

Customer
→ Sales Order

Sales Order
→ Manufacturing Order

Manufacturing Order
→ BoM

BoM
→ Components

Purchase Order
→ Vendors

PostgreSQL handles these relationships very efficiently.

# Database Design Philosophy

Inventory is the center of the ERP.

Every module affects inventory.

Sales:
Decreases stock

Purchase:
Increases stock

Manufacturing:
Consumes materials
Produces finished goods

Procurement:
Replenishes stock
